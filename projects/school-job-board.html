School Advisor Job Board

Centralized bulletin board for jobs, resources, and opportunities

The Problem

Advisors shared jobs and resources across multiple Google Classrooms, creating duplicate work. Students struggled to find current items or sort by deadlines.

Specific Pain Points

  • Duplicate posting: Advisors posted the same job/resource in 5+ different Google Classrooms
  • No centralized hub: Students had to check multiple places to find all opportunities
  • Hard to find current items: Old posts mixed with new ones, making it hard to see what was still relevant
  • No filtering or search: Students couldn't filter by deadline, category, or keyword
  • No role management: Anyone could post, leading to spam and irrelevant content

The Solution

Firebase app with search, category, and date filters. Advisor role management allows posting in one location. Calendar view and deadline sorting help students prioritize. Expired posts hidden by default with option to show archive.

Technical Implementation

Architecture Decisions

I chose Firebase for its real-time database capabilities and built-in authentication. This allows instant updates across all users and simple role-based access control.

Frontend

  • Vanilla JavaScript for filtering and search functionality
  • Responsive CSS Grid for card-based layout
  • Calendar library integration for deadline visualization
  • Firebase Authentication for user login and role verification

Backend (Firebase)

  • Firestore database for storing posts with timestamps and categories
  • Security rules ensuring only advisors can create/edit posts
  • Cloud Functions for auto-archiving expired posts
  • Real-time listeners for instant board updates

Interesting Challenges

Challenge: Role-Based Access Control

How do you ensure only advisors can post while still allowing all students to view? Google Classroom doesn't have this granularity.

Solution: Firebase Authentication with custom claims. Advisors get a special "advisor" role in their user profile, which Firebase security rules check before allowing write access.

Challenge: Auto-Archiving Old Posts

Posts with past deadlines should be hidden automatically, but advisors might want to reference them later.

Solution: Cloud Function runs daily to mark posts as "archived" if their deadline has passed. Students see active posts by default, but can toggle to view archived content.

Challenge: Real-Time Updates

When an advisor posts a new job, all 350+ students should see it immediately without refreshing.

Solution: Firestore real-time listeners automatically push new posts to all connected clients. No polling or manual refresh needed.

Development Process

What I Built First

Started with basic CRUD (create, read, update, delete) for posts. Once advisors could post and students could view, I added filtering and search based on their feedback.

User Testing & Iteration

  • Round 1: Advisors wanted bulk posting → added CSV import for multiple opportunities
  • Round 2: Students missed deadlines → added calendar view and deadline sorting
  • Round 3: Too many old posts cluttering the board → implemented auto-archiving
  • Round 4: Students wanted to search by keyword → added full-text search

What I'd Do Differently

If I rebuilt this today, I'd add email/SMS notifications for new high-priority posts (like scholarship deadlines). I'd also implement tagging so students could subscribe to specific categories and get alerts.

Lessons Learned

Solve real organizational problems

This wasn't a "cool tech project"—it solved a specific pain point advisors and students felt daily. Focusing on real needs led to high adoption.

Real-time updates matter

Firebase's real-time listeners mean students see new posts instantly. This simple feature dramatically improved the user experience.

Iterate based on usage patterns

The auto-archiving feature came from observing students struggle with old posts. Watching how people use your product reveals what to build next.
